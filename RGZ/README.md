Neon Blade Media

**Neon Blade Media** — учебная fullstack-видеоплатформа, разработанная в рамках расчётно-графического задания по web-технологиям.

Проект представляет собой сайт для просмотра, загрузки и удаления видеозаписей.  
Пользователь может зарегистрироваться, войти в аккаунт, загрузить видео с превью-картинкой, посмотреть видео в плеере и удалить свои загруженные записи.

Проект состоит из двух независимых частей:

- **backend** — серверная часть на Django и Django REST Framework;
- **frontend** — клиентская часть на React и Vite.

Backend отвечает за хранение пользователей, авторизацию, работу с видео и API.  
Frontend отвечает за интерфейс сайта, формы входа/регистрации, загрузку видео, вывод карточек и просмотр видео.

---

## Основная идея проекта

Сайт имитирует видеоплатформу для турнира или медиа-раздела события **Neon Blade**.

На главной странице пользователь видит:

- шапку сайта с логотипом;
- главный hero-блок;
- форму входа или загрузки видео;
- список загруженных видео;
- карточки видео с названием, описанием, автором и превью;
- модальное окно для просмотра видео;
- информационный блок о платформе;
- футер.

Если пользователь не авторизован, он может:

- смотреть список видео;
- открывать видео;
- зарегистрироваться;
- войти в аккаунт.

Если пользователь авторизован, он может:

- загружать видео;
- добавлять превью-картинку;
- смотреть видео;
- удалять свои видео.

## Используемые технологии

### Backend

В backend используются:

- **Python** — язык программирования серверной части;
- **Django** — основной web-фреймворк;
- **Django REST Framework** — создание REST API;
- **Simple JWT** — авторизация через JWT-токены;
- **Django CORS Headers** — разрешение запросов с frontend;
- **SQLite** — база данных для разработки;
- **Pillow** — работа с изображениями, используется для загрузки превью;
- **Django media files** — хранение загруженных видео и изображений.

### Frontend

Во frontend используются:

- **React** — библиотека для создания пользовательского интерфейса;
- **Vite** — инструмент для сборки и запуска frontend;
- **Axios** — отправка HTTP-запросов к backend;
- **CSS** — стилизация интерфейса;
- **LocalStorage** — хранение JWT-токена пользователя в браузере.

## Возможности проекта

В проекте реализованы следующие функции:

### Пользователи

- регистрация пользователя;
- вход пользователя;
- получение JWT-токена;
- сохранение токена в браузере;

- получение списка видео;

- загрузка превью-картинки;

- просмотр видео в модальном окне;
- удаление видео автором;
- защита удаления через permissions на backend.

### Интерфейс

- тёмный neon/cyberpunk-дизайн;
- шапка сайта с логотипом;
- hero-блок;
- формы входа, регистрации и загрузки;

- модальное окно просмотра;

# Запуск проекта

Для запуска проекта нужны два терминала:

- один для backend;
- второй для frontend.

---

## 1. Запуск backend

Перейти в папку backend:

```bash
cd backend
```

Создать виртуальное окружение:

```bash
python3 -m venv venv
```

Активировать виртуальное окружение:

```bash
source venv/bin/activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Применить миграции:

```bash
python3 manage.py migrate
```

Создать администратора:

```bash
python3 manage.py createsuperuser
```

Запустить backend:

```bash
python3 manage.py runserver
```

```bash
cd backend
source venv/bin/activate
python3 manage.py runserver 0.0.0.0:8000
```



## 2. Запуск frontend

Во втором терминале перейти в папку frontend:

```bash
cd frontend
```

Установить зависимости:

```bash
npm install
```

```bash
npm run dev -- --host 0.0.0.0
```

Должен открыться список видео в формате JSON или интерфейс Django REST Framework.

На сайте нужно открыть форму регистрации, ввести логин и пароль, затем зарегистрироваться.


После регистрации нужно войти под созданным логином и паролем.


После входа должна появиться форма загрузки видео.  
Нужно выбрать видеофайл, можно выбрать превью-картинку, затем нажать кнопку загрузки.

После загрузки видео появится карточка.  
При нажатии на кнопку просмотра откроется модальное окно с видеоплеером.


Если пользователь является автором видео, он может удалить его через кнопку удаления.
